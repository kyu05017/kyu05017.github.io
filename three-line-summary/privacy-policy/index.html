세줄요약(이하 “앱”)은 사용자가 음성을 기록하고, 기록된 내용을 바탕으로 요약문을 생성·저장·공유할 수 있도록 돕는 서비스입니다. 본 개인정보처리방침은 앱이 어떤 정보를 처리하는지, 어떤 목적으로 사용하는지, 사용자가 어떤 선택을 할 수 있는지를 설명합니다.

1. 앱이 제공하는 주요 기능

  • 마이크를 이용한 음성 기록 및 텍스트화
  • 텍스트 기반 요약 생성
  • 요약, 원문, 세션 정보의 로컬 저장
  • 저장된 히스토리 조회, 복원, 삭제, 공유

2. 처리할 수 있는 정보

2-1. 사용자가 직접 입력하거나 생성하는 정보

  • 음성 기록을 통해 생성된 텍스트(원문)
  • 요약 결과 텍스트
  • 세션 이름, 요약 상황, 요약 방식, 사용자 편집 내용
  • 사용자가 공유 기능을 통해 내보내는 텍스트

2-2. 앱이 기능 제공 과정에서 처리하는 정보

  • 기기 식별값(서버 인증 세션 발급 및 유지 목적)
  • 서버 접근 토큰(요약 서버 인증 세션 유지 목적)
  • 앱 언어, 테마, 요약 설정, 원문 저장 여부 등 설정값
  • 오류 대응 및 서비스 안정성 유지를 위한 최소한의 요청/응답 메타데이터
현재 첫 공개 배포 기준으로 앱 내 광고는 기본 비활성화(OFF) 상태를 전제로 준비하고 있습니다. 향후 광고 기능이 활성화될 경우, 본 정책과 스토어 고지 내용을 함께 업데이트할 수 있습니다.

3. 정보 이용 목적

  • 음성 기록 내용을 텍스트로 표시하고 요약 결과를 생성하기 위해
  • 사용자가 생성한 히스토리와 설정을 기기에 저장하고 다시 불러오기 위해
  • 요약 요청 시 서버와 통신하고 인증 세션을 유지하기 위해
  • 사용자가 요약 결과를 검토·편집·공유할 수 있도록 하기 위해
  • 앱 안정성 점검, 오류 대응, 서비스 운영을 위해

4. 저장 위치 및 처리 방식

4-1. 로컬 저장

앱은 일부 정보(예: 히스토리, 설정값, 서버 세션 정보)를 사용자의 기기에 로컬로 저장할 수 있습니다. 가능한 경우 보안 저장소를 우선 사용하며, 과거 저장값이 남아 있는 경우 이를 보안 저장소로 이전할 수 있습니다.

4-2. 서버 전송

사용자가 요약 생성을 요청하면, 요약 생성에 필요한 텍스트(예: 원문, 세션명, 날짜, 요약 상황/방식)가 운영 서버로 전송될 수 있습니다. 서버는 요약 생성 요청을 처리하고 결과를 앱에 반환합니다.

5. 보관 및 삭제

  • 사용자가 생성한 히스토리는 앱 내 로컬 저장소에 보관될 수 있습니다.
  • 사용자는 앱 내 기능을 통해 히스토리 삭제, 휴지통 이동, 복구 등을 수행할 수 있습니다.
  • 삭제된 일부 항목은 일정 기간 동안 휴지통에 남을 수 있으며, 이후 영구 삭제될 수 있습니다.
  • 사용자가 앱을 삭제하거나 앱 내 데이터 삭제 기능을 실행할 경우 일부 로컬 정보는 제거될 수 있습니다.

6. 제3자 제공 및 외부 서비스

앱은 원칙적으로 사용자의 정보를 판매하지 않습니다.

다만 아래와 같은 범위의 외부 서비스 연동이 있을 수 있습니다.

  • 요약 서버 운영 인프라: 요약 요청 처리, 인증 세션 발급, 오류 대응
  • 플랫폼 기본 공유 기능: 사용자가 직접 선택한 외부 앱으로 텍스트를 공유할 때
